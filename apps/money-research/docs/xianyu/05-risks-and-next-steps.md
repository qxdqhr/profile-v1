# 风险、合规与后续调研计划

## 1. 合规与法律风险

### 1.1 平台服务条款

- 闲鱼用户协议 generally 禁止未经授权的自动化操作
- 非官方 API 调用可能构成「未授权访问」
- 批量发品可能触发「垃圾信息/营销号」判定

### 1.2 风险等级

| 行为 | 风险 | 可能后果 |
|------|------|----------|
| 个人低频发品（<10/天） | 低-中 | 限流、滑块 |
| 个人中频（10-30/天） | 中 | 419 风控、短期禁发 |
| 高频批量（>50/天） | 高 | 封号、商品删除 |
| 多账号矩阵 | 很高 | 关联封号 |
| 对外 SaaS 服务 | 很高 | 法律诉讼风险 |
| 爬取用户数据 | 高 | 隐私合规问题 |
| 官方 ISV API | 低 | 正常商业合作 |

### 1.3 建议

- **自用工具** vs **对外服务**：风险差异巨大，调研阶段仅限自用 PoC
- 使用**测试账号**，避免主账号被封
- 遵守《网络安全法》《数据安全法》对个人信息的处理要求
- Cookie/Token 加密存储，不泄露他人账号

## 2. 技术风险

### 2.1 接口变更

| 变更类型 | 影响 | 应对 |
|----------|------|------|
| Mtop API 重命名 | 发品失败 | 抓包对比 + 版本锁定 |
| 签名算法升级 | 全部失败 | 关注开源社区更新 |
| 新增风控字段 | 部分失败 | 抓包补充参数 |
| Cookie 策略变化 | 登录失效 | 多方式登录备份 |

### 2.2 风控对抗

**已知风控机制：**

- `_m_h5_tk` 时效校验
- `x5sec` 滑块（阿里系通用）
- 设备指纹（utdid、umid、deviceId）
- IP 频率限制
- 行为序列分析（发布间隔、操作路径）
- 内容审核（违禁词、外联）

**419 滑块处理选项：**

| 方案 | 成本 | 可靠性 |
|------|------|--------|
| 人工浏览器过滑块 | 低 | 高 |
| Playwright 半自动 | 中 | 中 |
| 打码平台 | 中-高 | 中 |
| x5sec 参数复用 | 低 | 低（时效短） |

### 2.3 账号安全

- Cookie 泄露 = 账号被盗
- 建议使用独立测试账号
- 不在代码仓库中提交 Cookie
- 环境变量 / 加密 vault 存储

## 3. 业务风险

| 场景 | 风险 |
|------|------|
| 自动定价错误 | 低价卖出 |
| 类目识别错误 | 商品降权/下架 |
| 违禁内容 | 账号处罚 |
| IM 自动回复不当 | 买家投诉 |
| 重复发布 | 垃圾信息判定 |

**缓解措施：**

- 发布前人工确认（Human-in-the-loop）
- 价格/标题/template 校验规则
- goofish-cli 风格的风控护栏
- 发布日志与回滚（保留下架能力）

## 6. 决策树

```
是否需要企业级合规？
├─ 是 → 申请 ISV / 同城服务准入 → 官方 TOP API
└─ 否 → 是否需 IM 自动客服？
    ├─ 是 → myfish 框架 + WS 协议
    └─ 否 → 是否仅低频发品？
        ├─ 是 (<10/天) → Web Mtop (XianYuApis/goofish-cli)
        └─ 否 → 评估风控承受能力
            ├─ 可接受封号风险 → Web Mtop + 限速 + 多账号
            └─ 不可接受 → Playwright 半自动 + 人工确认
```
