## 2. 业务目标

在实验田提供可演示的**演出票务预定最小闭环**：用户能选择演出、进入选座页完成座位点选、查看订单摘要并执行**模拟支付**（不产生真实资金与外部票务接口）。

## 3. 用户与场景

| 角色 | 场景 |
|------|------|
| 访客用户 | 浏览演出列表，查看时间、场馆、起价 |
| 访客用户 | 选择一场演出后进入选座；区分可选 / 已选 / 已售 |
| 访客用户 | 确认座位后查看金额明细（含模拟服务费），点击模拟支付完成流程 |

## 4. 功能范围

### 4.1 本期必须（MVP）

- **演出列表**：展示多条内置演示演出（标题、日期、场馆、单价区间或起价）。
- **购票入口**：从列表进入选座流程（单页分步或同 URL 状态机均可）。
- **选座页**：二维座位矩阵（行/列），**已售**不可点；**可选**可切换为**已选**；展示图例与已选数量、合计金额。
- **订单确认**：列出座位明细、小计、服务费、总计；**模拟支付**按钮，成功后给出明确反馈并可返回列表。

### 4.2 本期不做（非范围）

- 真实支付、对账、发票、退改签规则引擎。
- 登录鉴权、库存与锁座服务端一致性（并发抢票）。
- 与 `sa2kit/showmasterpiece` 画集数据的耦合；后台排期管理。
- 打印票、二维码闸机、短信通知。

## 5. 业务规则（MVP 级）

- 单用户会话内选座；刷新页面可接受状态丢失（无持久化）。
- 同一座位不可重复售出：已售座位由前端**固定种子**模拟，仅用于演示。
- 建议单订单可选座位上限（如 6），超出给出提示（可在实现中落地）。

## 6. 验收标准

1. 从实验田入口进入 `/testField/TicketBooking` 可见演出列表。
2. 任选一场可进入选座；已售格与可选/已选视觉区分清晰。
3. 至少选择 1 个可选座位后可进入确认页；金额计算正确。
4. 模拟支付后显示成功态，且可重新开始购票流程。
5. 布局在移动端与桌面端均可正常使用（Tailwind 响应式）。

## 7. 假设与风险

| 假设 | 风险 |
|------|------|
| 无后端锁座 | 多标签页不互斥，仅教学/演示用途 |
| 演示数据写死在前端 | 与真实库存不一致，迭代需接 API + DB |
