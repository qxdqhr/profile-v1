# 04 — 接口与数据字典草案（MVP → 扩展）

## 1. MVP 阶段：前端本地契约

无 HTTP 接口。以下为 **TypeScript 领域模型**（实现于 `src/modules/ticketBooking/types`），供组件与工具函数共用。

### 1.1 Performance（演出）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 稳定主键，如 `perf-sw-lake` |
| title | string | 演出名称 |
| subtitle | string? | 副标题 |
| startsAt | string | ISO 日期时间（展示用） |
| venue | string | 场馆名称 |
| city | string? | 城市 |
| currency | `'CNY'` | 货币 |
| basePrice | number | 单座基准价（演示统一价） |
| posterGradient | string | Tailwind gradient 类名（无真实海报图 MVP） |

### 1.2 Seat（座位）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 如 `A-4` |
| rowLabel | string | 如 `A` |
| col | number | 1-based 列号 |
| status | `'available' \| 'sold'` | 初始态；`selected` 由 UI 状态派生 |

### 1.3 OrderDraft（确认前草稿）

| 字段 | 类型 | 说明 |
|------|------|------|
| performanceId | string |  |
| seatIds | string[] | 已选座位 |
| subtotalCents | number | 可选：用「元」整数演示则省略分 |
| serviceFee | number | 元 |
| total | number | 元 |

> MVP 实现可采用「元」为单位 number，与 PRD 中 ¥ 展示一致即可。
