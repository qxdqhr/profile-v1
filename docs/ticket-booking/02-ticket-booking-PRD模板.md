# 02 — 票务预定系统 PRD（MVP 骨架）

## 1. 产品概述

**一句话**：实验田内的演出票务演示：列表 → 选座 → 订单确认（模拟支付）。

**价值**：验证信息架构与交互（与画集类业务同在 `(cyhj)` 下），为后续接入真实库存与支付预留扩展点。

## 2. 用户故事

| ID | 作为 | 我想要 | 以便 |
|----|------|--------|------|
| US-1 | 访客 | 浏览演出列表 | 选择感兴趣的场次 |
| US-2 | 访客 | 在平面图上点选座位 | 购买指定区域座位 |
| US-3 | 访客 | 看到费用明细并确认 | 完成下单决策（模拟） |

## 3. 信息架构与导航

```
/testField/TicketBooking
├── 步骤 1：演出列表（卡片 / 行布局，移动端优先）
├── 步骤 2：选座（舞台示意 + 网格 + 图例 + 底部操作栏）
└── 步骤 3：订单确认（明细 + 模拟支付）
```

返回行为：步骤 2 → 1、步骤 3 → 2 或取消回 1。

## 4. 页面说明

### 4.1 演出列表

- 字段：名称、日期时间、场馆、价格展示（起价或固定档）。
- 主操作：「选座购票」进入步骤 2。

### 4.2 选座

- 展示当前演出摘要条（名称 + 时间）。
- 座位状态：`available` | `selected` | `sold`。
- 底部：已选个数、合计金额、「确认选座」（至少 1 座且全部有效）。

### 4.3 订单确认

- 座位列表（如「A 排 4 号」）。
- 价格：小计、服务费（固定或比例，与实现一致）、总计。
- 「模拟支付」→ 成功 Toast/卡片 + 「返回演出列表」。

## 5. 非功能

- **性能**：纯前端状态，首屏无远程阻塞。
- **可访问性**：按钮与座位格具备可读标签（`aria-label`）。
- **国际化**：本期仅中文。

## 6. 数据与扩展（预留）

后续版本可引入：`performances` 表、`shows` 场次、`seat_lock`、订单表；API 置于 `src/modules/ticketBooking/api` 并由 `src/app/api/ticketBooking/**` 转发（参照 `ideaList`）。
